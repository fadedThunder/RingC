PROJECT DETAILS

This project consists of Zendesk interview task for an SDET position. It consists of testcases to test various API listed below.
This is a Maven project and the testing framework used is TestNG. The Project details are given below.
Project Name: testInterview.
Project has 2 parts: 
1)	src/main/java, consists of commonly used classes across the projects. It has TestBase and TestConstant class.
2)	src/test/java, consists of various test classes 
•	 TestTicketCreate : Consists of test case to create the ticket and asserting the status code returned in the response. (POST)
•	 TestTicketComment : Consists of test case to comment on the ticket based on the author id and asserting the code in the response . (POST)
•	TestTicketList : Consists of test cases to get the list of tickets based on the Organization id, 
•	TestTicketDelete

For Authorization: Zendesk Account details: SaonliTest Test
The auth is taking place with Email and Password created.

DEPENDENCIES

1.	Eclipse
2.	Java 1.8 in the build path of Eclipse
3.	Install Maven
4.	TestNG configured to run the test as TestNG test case.
5.	All the other dependencies are there in the POM with the project.

HOW TO INSTAL AND EXECUTE
 
1.	Unzip the project in desktop/ local.
2.	Import the project in Eclipse as Maven Project.
3.	Compile the project (Can be done from eclipse)
4.	Run the test from the test class: Right click anywhere on the class > Run As > TestNG test.
TEST RESULT AND TEST DATA
There are various ways of providing test data during run time that is test cases are parameterized.
Usually when used with  Maven and Jenkins : System.getProperty(“Parameter Name”) being widely used, however for easy to execute in the project Scanner has been used. Detailed description has been given  in  “How to Test” 
The test cases are parameterized so any test cases from any organization id , or any author id can be tested.
For Domain name change: Change the 
public static String TESTINSTANCE = "https://shalutest.zendesk.com";
Change the domain name like shalutest to desired name in the TestConstant class.
